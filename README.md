# QuickNotes — дизайн-документ (README.md)

**Краткое описание:**

QuickNotes — мобильное приложение, разработанное на **Flutter**, для создания и управления заметками. Пользователь может добавлять, редактировать, удалять заметки, хранить их локально и при желании синхронизировать с публичным API цитат для вдохновения. Приложение выполнено в минималистичном дизайне с акцентом на простоту и удобство.

---

## 1. Цель лабораторной работы

Разработать концепцию и структуру Flutter-приложения, подготовив дизайн-документ, который послужит основой для следующих этапов разработки:
- **ЛР4 (верстка)** — макеты экранов и интерфейс (Flutter Widgets);
- **ЛР5 (логика)** — обработка действий пользователя и навигация;
- **ЛР6 (данные)** — хранение заметок и работа с API.

---

## 2. Тема приложения

**Тема:** Приложение заметок — *QuickNotes*

**Задача:** обеспечить возможность быстро сохранять и редактировать текстовые заметки с простым интерфейсом.

**Почему Flutter:** кроссплатформенность (Android/iOS), простая реализация UI, наличие удобных пакетов (Provider, Hive, etc.).

---

## 3. Основные функции (MVP)

- Добавление новой заметки (заголовок + текст).
- Просмотр списка заметок.
- Редактирование и удаление заметок.
- Хранение данных локально (Hive или SharedPreferences).
- Загрузка цитаты дня с публичного API (например, https://api.quotable.io/random).
- Навигация между экранами (главный экран ↔ экран редактирования ↔ экран о приложении).
- Простая стилизация с использованием Material Design.

---

## 4. Структура экранов

1. **HomeScreen (Главный экран)**
   - AppBar: заголовок приложения, кнопка меню.
   - Список заметок (ListView / GridView с карточками заметок).
   - FloatingActionButton для добавления новой заметки.
   - Внизу отображается мотивационная цитата.

2. **NoteEditorScreen (Редактор заметки)**
   - Поле для заголовка.
   - Многострочное поле для текста заметки.
   - Кнопка сохранения.

3. **AboutScreen (О приложении)**
   - Краткое описание приложения.
   - Кнопка для загрузки новой цитаты.
   - Отображение текущей цитаты.

---

## 5. Навигация

- HomeScreen → NoteEditorScreen (при добавлении/редактировании заметки)
- HomeScreen → AboutScreen (через меню)
- NoteEditorScreen → HomeScreen (при сохранении/отмене)

Реализация: `Navigator.push()` / `Navigator.pop()` или `go_router` для более структурированной навигации.

---

## 6. Сценарии использования

### Сценарий 1 — Добавление заметки
1. Пользователь нажимает кнопку "+".
2. Открывается NoteEditorScreen.
3. Вводится заголовок и текст заметки.
4. При нажатии "Сохранить" заметка появляется в списке.

### Сценарий 2 — Редактирование
1. Нажать на существующую заметку.
2. Внести изменения.
3. Сохранить — данные обновляются на главном экране.

### Сценарий 3 — Удаление
1. Долгое нажатие или кнопка в карточке заметки.
2. Подтверждение удаления.
3. Заметка исчезает из списка.

### Сценарий 4 — Загрузка цитаты
1. При открытии главного экрана или экрана «О приложении» отправляется запрос к API.
2. Полученная цитата отображается внизу экрана.
3. При отсутствии сети выводится заглушка («Нет соединения»).
